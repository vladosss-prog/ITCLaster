# EventHub

Веб-платформа для организации и проведения мероприятий — форумов, конференций, хакатонов.
Заменяет связку «сайт + мессенджеры + таблицы + файл программы» одним инструментом.

Проект разработан в рамках **Цифрового хакатона 2026** — XI Международный ИТ-Форум «Основы цифрового будущего», ОмГТУ × ИТ-Кластер Сибири.

---

## Что умеет платформа

### Для организатора
- Создание мероприятий, секций и докладов
- Назначение кураторов на секции
- Канбан-доска задач по каждому мероприятию
- Автоматический подсчёт % готовности мероприятия
- Календарь с дедлайнами задач
- WebSocket-уведомления при смене статуса задач
- Чат команды организаторов
- Видимость всего без вмешательства в операционку секций

### Для куратора секции
- Определение формата секции (доклады / круглый стол / деловая игра)
- Поиск и приглашение спикеров по ФИО
- Согласование зала, времени и технического обеспечения
- Назначение модератора секции
- Контроль подтверждения данных спикеров
- Управление порядком выступлений
- Итоговый отчёт о проведении секции

### Для спикера
- Личный кабинет с данными доклада
- Подтверждение участия, загрузка bio и фото
- Указание формата выступления (очный / онлайн)
- Просмотр вопросов и комментариев к докладу

### Для участника
- Публичная программа мероприятия
- Личное расписание — отмечаешь какие доклады посетишь
- Комментарии и вопросы к докладам
- Оценки докладов (1–5 звёзд)
- Мессенджер: личные сообщения и групповые чаты мероприятий

## Структура базы данных
```
users               — аккаунты пользователей
events              — мероприятия
event_memberships   — кто в каком мероприятии и в какой роли
sections            — секции внутри мероприятия
reports             — доклады внутри секции
tasks               — задачи канбан-доски
section_reports     — итоговые отчёты кураторов
chat_rooms          — комнаты чата (групповые и личные)
chat_messages       — сообщения
```

## Роли пользователей

| Роль | Уровень | Описание |
|---|---|---|
| `ORGANIZER` | глобальная | создаёт мероприятия, видит всё |
| `PARTICIPANT` | глобальная | базовый аккаунт |
| `CURATOR` | контекстная | управляет своей секцией |
| `SPEAKER` | контекстная | управляет своим докладом |

Один человек может быть куратором на одном мероприятии и участником на другом одновременно.

## Быстрый старт

### Требования
- Python 3.11
- PostgreSQL 18
- Node.js 18+ (для фронтенда)
- Git

### 1. Клонируй репозиторий
```bash
git clone https://github.com/ВАШ_НИК/eventhub.git
cd eventhub
```

### 2. Создай виртуальное окружение
```bash
py -3.11 -m venv venv
venv\Scripts\activate
```

### 3. Установи зависимости
```bash
pip install -r requirements.txt
```

### 4. Создай базу данных
Открой **pgAdmin** → правой кнопкой на **Databases** → **Create** → **Database**
Имя: `eventhub_db` → Сохранить

### 5. Настрой переменные окружения
```bash
copy .env.example .env
```
Открой `.env` и укажи свой пароль от PostgreSQL:
```
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost/eventhub_db
SECRET_KEY=придумайте_длинную_строку
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=480
```

### 6. Создай таблицы в БД
```bash
cd backend
..\venv\Scripts\alembic upgrade head
```

### 7. Запусти сервер
```bash
..\venv\Scripts\uvicorn main:app --reload
```

- API: [http://localhost:8000](http://localhost:8000)
- Swagger документация: [http://localhost:8000/docs](http://localhost:8000/docs)

## Разработка

### Git-процесс
```bash
# Создай свою ветку
git checkout -b feature/tasks

# Работаешь, коммитишь
git add .
git commit -m "feat: CRUD задач с фильтрами"
git push origin feature/tasks

# Каждое утро забираешь изменения команды
git pull origin main
```

### Распределение команды

| Человек | Направление | Файлы |
|---|---|---|
| Бек 1 | Ядро, Auth, Events | `models.py`, `routers/auth.py`, `routers/events.py` |
| Бек 4 | Задачи, Календарь | `routers/tasks.py` |
| Бек 5 | Участники, Программа | `routers/participants.py` |
| Бек 6 | Мессенджер, WebSocket | `routers/chat.py` |
| Фронт | React UI | `frontend/` |
| Дизайн | Figma | — |

### Правила
- `models.py` редактирует только **Бек 1** после согласования с командой
- Если нужна новая таблица — обсуждаем голосом, Бек 1 добавляет и делает новую миграцию
- После `git pull` всегда запускай `alembic upgrade head` — могли появиться новые миграции
- `.env` никогда не пушить в гит
